**HomePage implementation:**

State:
- `selectedSpool: SpoolCardData | null` — card being operated on
- `selectedOperation: Operation | null` — from OperationModal
- `selectedAction: Action | null` — from ActionModal

Handlers:
- `handleAddSpool(tag)`: calls addSpool(tag), closes add-spool modal, enqueue success toast
- `handleCardClick(spool)`: setSelectedSpool, push('operation')
- `handleSelectOperation(op)`: setSelectedOperation, push('action')
- `handleSelectMet()`: push('metrologia')
- `handleSelectAction(action)`: setSelectedAction, push('worker')
- `handleWorkerComplete()`: clear modals, refreshSingle(selectedSpool.tag_spool), enqueue success toast, reset selected states
- `handleMetComplete(resultado)`: clear modals, if APROBADO call removeSpool + enqueue "aprobado" toast; if RECHAZADO call refreshSingle + enqueue "rechazada" toast; reset selected states
- `handleCancel()`: CANCELAR dual logic per research patterns:
  - If !selectedSpool.ocupado_por: removeSpool (frontend-only, STATE-03)
  - If operacion_actual is 'ARM' or 'SOLD': call finalizarSpool with selected_unions=[] (zero-union CANCELAR path)
  - If operacion_actual is 'REPARACION': call cancelarReparacion
  - Parse worker_id from ocupado_por string "MR(93)" -> 93
  - On success: removeSpool + clear modals + toast
  - On error: enqueue error toast
- `handleRemoveCard(tag)`: removeSpool(tag) — for the X button on cards
- `handleModalClose()`: pop() from stack; if stack becomes empty, reset selected states

function parseWorkerIdFromOcupadoPor(ocupadoPor: string): number | null {

}
